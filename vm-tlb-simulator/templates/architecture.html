Architecture

How the simulator translates virtual addresses

Project Overview

A Python simulator that reproduces how a CPU's MMU translates virtual addresses into physical ones — modeled after the x86-64 architecture. Implements segmentation, multi-level paging, TLB caching with three replacement policies, and page fault handling with swap.

Every translation produces a step-by-step trace, so you can see exactly what happens inside the MMU on each memory access.

Four-Stage Translation Pipeline

1. Segmentation. The logical address is split into a 16-bit segment selector and a 48-bit offset. We check the offset against the segment's limit and add the base — producing a linear address.

2. TLB lookup. A small cache of recent VPN → frame_number translations. On a hit, we skip the page table walk entirely. The TLB uses one of three replacement policies (FIFO, LRU, Clock) when full.

3. Page table walk. Two implementations available:

  • Single-level — a flat sparse dictionary mapping VPN to PTE.
  • Multi-level — a four-level sparse tree (PGD → PUD → PMD → PT), 9 bits per level, mirroring real x86-64 hardware. Intermediate tables are allocated lazily.

4. Page fault handler. If the PTE is missing or invalid, the handler allocates a free physical frame. If memory is full, it evicts an existing page via the replacement policy, writes it to swap if dirty, and retries the translation.

Key Design Decisions

Skeleton before code. We agreed on shared types and module interfaces before anyone wrote a line of implementation. Without this, six people would have produced incompatible PTE definitions.

Two abstract interfaces. Translator is implemented by segmentation, single-level, and multi-level paging — so switching modes in main.py is a one-line change. ReplacementPolicy is implemented by FIFO, LRU, and Clock, and used by both the TLB and the page fault handler — eliminating duplicated code.

x86-64 as the reference. 4 KB pages, 48-bit virtual addresses, 4 levels of 9-bit indices each. Keeping these parameters fixed across all modules meant no one had to second-guess address layouts.

AI assistance with boundaries. We used Claude (claude.ai) for explanations, code generation, and debugging. But the architectural decisions stayed with the team. The AI was a tool, not a designer.
